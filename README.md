UAM-RL-Scheduler-IncomingAware

UAM 垂直起降场调度仿真研究
城市平均地面速度 20 km/h | 单乘客 eVTOL 容量 | 8 起降场 | X 型布局
排队预测强化学习 (PPO) | 最优 ATT: 35.68 分钟


一、项目摘要

UAGMC 论文（IEEE TITS 2026）首次提出了端到端的空地协同空中出租车调度框架，
通过 MSCE 多源嵌入与 STIN 时空 Transformer 网络处理高维异构状态信息，验证
了深度强化学习在 UAM 起降场选择中的有效性。然而，真实城市交通环境呈现车速
低（市区平均 20-30 km/h）、起降场数量多、乘客出行距离分布及需求差异大等
特征，现有方法在贴近实际场景下的性能表现与计算开销仍需进一步探索。

基于上述问题，本项目在 UAGMC 框架基础上构建了一个符合未来城市交通条件的
UAM 起降场调度仿真平台，核心思路为：

(1) X 型 8 起降场网络，覆盖 40x40 km 城市区域，增强对多方向出行需求的空间感知能力
(2) 出行距离感知的乘客分类机制，短途乘客直接分配地面交通
(3) 真实 eVTOL 调度动力学仿真，多机协同的全生命周期调度
(4) 排队预测强化学习，状态中加入"飞往起降场的飞机数量"特征

实验结果表明，在 20 km/h 城市平均速度、单乘客 eVTOL 容量、8 起降场的场景
下，轻量级 PPO+MLP 模型达到平均总出行时间（ATT）35.68 分钟。


二、问题建模

2.1 出行链建模

每位乘客的出行包含四个阶段：
起点 -> CAV -> 起飞机场 -> 排队 -> eVTOL -> 飞行 -> 降落机场 -> CAV -> 终点

总出行时间：
TT_i^tot = TT_o_i->k_i^g + TW_k_i^wait + TT_k_i->l_i^a + TT_l_i->d_i^g

其中：TT^g = 地面出行时间，TW^wait = 排队等待时间，TT^a = 空中出行时间

2.2 起降场布局

8 个垂直起降场呈 X 型布局，覆盖 40x40 km 城市区域：
编号0 (5,5)，编号1 (35,5)，编号2 (15,15)，编号3 (25,15)，
编号4 (15,25)，编号5 (25,25)，编号6 (5,35)，编号7 (35,35)

2.3 乘客分类机制

短途乘客（地面出行时间小于任何可行飞行方案）直接分配地面交通，不进入起降场排队。

2.4 eVTOL 调度策略

每个时间步（1 tick = 0.1 分钟）依次执行：
(1) 起飞阶段：排队乘客与空闲 eVTOL 匹配，立即起飞
(2) 调度阶段：定位排队最长的起降场，调度最近的空闲 eVTOL 前往支援
(3) 飞行更新：更新飞行中 eVTOL 剩余时间，到达后加入空闲池

2.5 优化目标

最小化所有乘客的平均总出行时间：
min (1/|N|) * sum_i TT_i^tot

该问题建模为马尔可夫决策过程（MDP），采用深度强化学习求解。


三、方法

3.1 状态空间设计（20维）

S(t) = [Q_0,...,Q_7, F_0,...,F_7, x_o, y_o, x_d, y_d]

Q_k: 起降场 k 排队人数（8维）
F_k: 飞往起降场 k 的 eVTOL 数量（8维）
(x_o,y_o): 乘客起点坐标（2维）
(x_d,y_d): 乘客终点坐标（2维）

F_k 的设计动机：使智能体能够预判排队演化趋势。

3.2 动作空间

七选一离散动作：A(t) 属于 {0,1,...,6}
动作 a 表示选择按总出行时间排序后的第 a 个起降场（0为最近，6为第七近）。
降落机场固定为离目的地最近的起降场。

3.3 奖励函数

R(t) = -(1/|N_t|) * sum_i TT_i^tot
直接对齐优化目标，无奖励塑形。

3.4 PPO 超参数

学习率: 5e-4（线性衰减）
n_steps: 2048
n_epochs: 10
batch_size: 64
gamma: 0.99
GAE lambda: 0.95
clip range: 0.2
熵系数: 0.03
网络结构: 256 x 256


四、实验设置

地图大小: 40 x 40 km
起降场数量: 8 个
地面 CAV 速度: 20 km/h
eVTOL 飞行速度: 120 km/h
eVTOL 载客量: 1 人
乘客规模: 800 人（固定随机种子）
eVTOL 数量: 8 / 12 / 16 架
训练步数: 12 万步

对比方法：
(1) 全部走地面：所有乘客全程 CAV
(2) 最短路径：不考虑排队的静态最短路径
(3) 最短路惩罚：最短路径 + 225 * 排队人数
(4) 基线 PPO（12维）：不含飞往 eVTOL 数量特征

评估指标：AGT（平均地面出行时间）、AWT（平均等待时间）、AAT（平均空中出行时间）、ATT（平均总出行时间）、系统完成时间

六、实验结论

(1) PPO 方法优于规则基线：8/12/16 架配置下相比最短路径分别降低 41.0%/40.8%/22.7%。

(2) 排队预判特征有效：20维状态相比12维状态，AWT 在 8/12/16 架配置下分别降低 7.3%/7.8%/10.8%。

(3) 运力紧张时模型优势更显著：8架时相比基线 PPO 降低 6.3%，16架时仅降低 4.8%。

(4) 地面-飞行分类机制有效：短途乘客被合理分配地面交通，有效缓解排队压力。


七、研究局限与未来方向

(1) 算力受限：训练步数仅12万步，未来将扩展至50-100万步。

(2) eVTOL 调度规则为人工设计：未来引入多智能体强化学习实现联合优化。

(3) 场景假设与真实运营存在差距：未来引入动态乘客生成、容量约束、充电约束。

(4) 算法未充分捕捉时序特征：未来引入 Transformer 或 LSTM。
